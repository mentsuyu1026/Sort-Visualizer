# sort_visualizer

ソートアルゴリズムの可視化（小規模、8要素想定）を行う簡易プログラムです。C言語で書かれたソースと実行バイナリが含まれており、色設定は `colors.txt` から読み込みます。

**主な特徴**
- **軽量**: 単一ソースファイル（`sort8.c`）をコンパイルして実行します。
- **色設定**: `colors.txt` で色や表示設定を変更できます。

**ファイル構成**
- `sort8.c`: ソースコード（C）。
- `sort8`: ビルド済みの実行ファイル（存在する場合）。
- `colors.txt`: 可視化に使う色設定ファイル。
- `README.md`: このファイル。

**ビルド方法**
必要ならソースからビルドします。Linux 環境で `gcc` を使う例:

```bash
gcc -O2 -o sort8 sort8.c
```

**実行方法**
ビルド済みのバイナリがある場合は、作業ディレクトリで直接実行します:

```bash
./sort8
```

`colors.txt` の内容を読み込む実装になっているため、色や表示を変更したい場合はファイルを編集してから再実行してください。

**例**
- `./sort8` を実行すると、コンソール上（または設定された出力）にソートの各ステップが表示されます。

**開発メモ / 注意点**
- 大きな配列や高負荷の可視化には最適化が必要です。

**実装されているソートアルゴリズム（一覧）**
本プログラムに実装されているアルゴリズムは以下の通りです（実装は `sort8.c` にあります）。

- `BubbleSort` — 隣接要素を比較して交換する典型的な O(n^2) の安定ソート。教育目的に適する可視化が容易なアルゴリズムです。
- `SelectionSort` — 最小（最大）要素を選んで前へ配置する O(n^2) の非安定ソート。交換回数が比較的少ない特徴があります。
- `InsertionSort` — 挿入によるソートで、ほぼ整列済みの配列に対して高速（平均 O(n^2)、最良 O(n)）。安定です。
- `ShellSort` — ギャップ挿入法による改良挿入ソート。ギャップ列に依存しますが実用上高速化されることが多い（平均はケース依存）。
- `QuickSort` — 反復実装（スタック式）での分割統治法。平均 O(n log n)、最悪 O(n^2)、通常は高速だがピボット選択に依存します（非安定）。
- `HeapSort` — ヒープ（最大ヒープ）を使った O(n log n) の安定でないソート（安定性は保証されません）。メモリはインプレース。
- `MergeSort` — 下位から上位へ合併するボトムアップ実装の O(n log n) の安定ソート（バッファを内部で使用）。
- `BinSort`（Counting/Bucket） — 値が 1..n の範囲に限定された線形時間近似の計数ソート的手法（O(n + k)、ここでは値範囲を配列長にクリップしている）。

各アルゴリズムは可視化用にステップ単位で状態を保持する実装になっており、比較や移動の回数が計測されます。詳細は実装を参照してください: [sort8.c](sort8.c)
